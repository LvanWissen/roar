hasLocationop back to ToC or Object Property ToC

IRI: https://w3id.org/roar#hasLocation

Use to indicate that a Person has a relation to a certain Location. Use a structured value to indicate a specific role of this person to the location or to specify a temporal validity in which this Person-Location relation is valid (use `sem` timestamp properties for this). Example: ```ttl :person1 a roar:PersonObservation ; roar:hasLocation [ rdf:value :location1 ; roar:role "inhabitant" ; sem:hasBeginTimeStamp "1782-11-20" ; sem:hasEarliestEndTimesStamp "1782-11-20" ] . ```
has domain
Entity c
has range
Location c

hasPersonop back to ToC or Object Property ToC

IRI: https://w3id.org/roar#hasPerson

Use to indicate that a Location is in relation to a certain Person. Use a structured value to indicate a specific role of this location to the person or to specify a temporal validity in which this Location-Person relation is valid (use `sem` timestamp properties for this). Example: ```ttl :location1 a roar:LocationObservation ; roar:hasLocation [ rdf:value :person1 ; roar:role "residence" ; sem:hasBeginTimeStamp "1782-11-20" ; sem:hasEarliestEndTimesStamp "1782-11-20" ] . ```
has domain
Entity c
has range
Person c
